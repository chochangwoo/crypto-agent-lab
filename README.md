# Crypto Agent Lab

크립토 리서치 + 트레이딩 멀티 에이전트 파이프라인

## 목표

외부 데이터 수집 → AI 리서치 → 자동 백테스트 → 사람 승인 → 실거래 반영까지의 파이프라인을 에이전트로 자동화한다.

## 연관 프로젝트

| 프로젝트 | 역할 | 상태 |
|----------|------|------|
| `upbit-quant` | 실거래 트레이딩 봇 (Railway 24/7) | 운영 중 |
| `salkkamalka-backtest` | 백테스트 웹 대시보드 | 개발 중 |
| **crypto-agent-lab** | 멀티 에이전트 리서치/자동화 | **신규** |

## 에이전트 구조

```
[1] 데이터 수집 에이전트 (1시간마다)
     │  공포탐욕지수, 펀딩비율, BTC 도미넌스, 뉴스 감성
     ▼
[2] 리서치 에이전트 (매일 09:00)
     │  시장 브리핑 + 전략 아이디어 제안
     ▼
[3] 퀀트 에이전트 (티켓 수신 시 + 주 1회)
     │  백테스트 검증 + 승인 요청
     ▼
[사람] 승인/거부
     ▼
[4] 트레이딩 에이전트 (= upbit-quant, 기존 봇)
     │
     ▼
[5] 모니터링 에이전트 (30분마다)
     성과 추적, 이상 감지, 전략 괴리 알림
```

## 구현 로드맵

### Phase 1: 데이터 기반 구축
- [ ] 공포탐욕지수 수집기
- [ ] 바이낸스 펀딩비율 수집기
- [ ] BTC 도미넌스 수집기
- [ ] 뉴스 감성 분석기
- [ ] Supabase market_signals 테이블
- [ ] 수집 스케줄러

### Phase 2: 리서치 자동화
- [ ] 일일 시장 브리핑 생성 (Claude API)
- [ ] 외부 데이터 ↔ 수익률 상관 분석
- [ ] 전략 아이디어 자동 제안

### Phase 3: 퀀트 자동화 + 오케스트레이션
- [ ] 백테스트 자동 실행 (제안 기반)
- [ ] 승인 요청 플로우 (텔레그램)
- [ ] Paperclip 도입 (에이전트 3개 이상 시점)
- [ ] 전략 자동 배포 파이프라인

### Phase 4: 모니터링 + 통합
- [ ] 실시간 성과 추적
- [ ] 이상 감지 (MDD, 슬리피지, API 에러)
- [ ] 전략 괴리 감지 (실전 vs 백테스트)
- [ ] upbit-quant 연동 완성

## 기술 스택

| 역할 | 도구 |
|------|------|
| 언어 | Python 3.13 |

| 오케스트레이션 | Paperclip (Phase 3~) |
